Assumptions


I’m assuming you’re starting out with at least one unconfigured switch that you probably got second or third-hand. I’m also assuming you’ll be using a Windows computer to follow along with the labs. If you use Linux, I’m assuming you know how to translate the Windows-only portions of this document into Linux.


Prerequisites


To get the most out of this book, you should have two switches: a layer-3 switch like a Cisco Catalyst 3560 and a layer-2 switch like a Catalyst 2960.


You’ll begin by configuring the layer-3 switch, which will be called “Switch1” throughout the book. (It’s fine if you have two layer-3 switches. Just pick one to be Switch1, and make sure you don’t get them mixed up.)


To get started, you need two things:


  1. A Cisco rollover cable
  2. A terminal emulator program like PuTTY


Let’s start with the first. A Cisco rollover cable is a blue cable with a trapezoid-shaped, 9-pin RS-232 serial plug on one end, and an RJ-45 connector on the other (it looks like one end of a network cable).


If your computer has a 9-pin serial port, you can plug the rollover cable directly. If not, you’ll need to get a USB-to-serial adapter.


Once you’ve got the rollover cable connected to your computer and the switch, you’ll need a program like PuTTY, which you can download at http://www.chiark.greenend.org.uk/~sgtatham/putty/download.html.


Lab Setup


1. Launch PuTTY, or the terminal emulator of your choice, and enter the serial line (aka COM port) the rollover cable is connected to. In the picture below, mine is connected to COM3. Click “Open”.




You’ll likely have to go through some trial-and-error. If you get an error from PuTTY, try a different COM port until you no longer get an error.


2. Plug in Switch1’s power cable. If all is well, after several seconds, you should see the switch’s boot sequence.




4. When it’s done booting it may ask you, “Would you like to enter the initial configuration dialog?” If you see this, type “no” and hit enter before moving on. Otherwise, just proceed to the next step.





5. On Switch1, follow the instructions exactly:

Type “enable” and hit enter

Type “configure terminal” and hit enter

Type “interface vlan1” and hit enter

Type “ip address 192.168.1.101 255.255.255.0” and hit enter

Type “no shutdown” and hit enter

Type “exit” enter

Type “int range fa0/1-20” and hit enter

Type “switchport mode access” and hit enter

Type “hostname Switch1” enter

Line vty 0 4

Transport input telnet

Login local

Exit

Username admin privilege 15 secret cisco

Enable secret cisco

Type “end” and enter

Type “copy running-config startup-config” and hit enter twice


6. Connect an Ethernet cable from the switch’s FastEthernet0/1 interface to the NIC on your computer.


7. Configure the computer’s interface with the IP address 192.168.1.10 and the subnet mask 255.255.255.0. Leave the gateway and DNS blank.




8. In PuTTY, or whatever terminal emulation program you’re using, you should see Vlan1 change state to up:


*Mar 1 00:09:14.369: %LINK-3-UPDOWN: Interface FastEthernet0/1, changed state to up

*Mar 1 00:09:15.376: %LINEPROTO-5-UPDOWN: Line protocol on Interface FastEthernet0/1, changed state to up

*Mar 1 00:09:42.395: %LINEPROTO-5-UPDOWN: Line protocol on Interface Vlan1, changed state to up


9. On the computer, open up a command prompt and enter:

ping 192.168.1.101


If you get a response, you’re good to go. If not, you may have missed a step. Go back over the instructions until you’re able to complete this step successfully.



Configuring Switch2


Repeat the process for Switch2, but change the following commands when you get to step 5:

hostname Switch2

ip address 192.168.1.102 255.255.255.0
